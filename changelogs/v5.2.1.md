## v5.2.1 - Workflow Center 1.0 正式完成

### Workflow Center 1.0
- **feat**: 新增完整工作流中心，支持工作流设置、模板 CRUD、AI 生成模板、运行列表、运行详情、步骤进度、事件流与删除实例
- **feat**: 支持复杂任务自动接管原会话，自动创建工作流实例、即时确认回复、原会话继续驱动、审批/暂停/恢复/重试/取消
- **feat**: 工作流状态、控制文案、进度提示与运行页主界面全面中文化，并加入自动刷新与消息中心同步

### 文件产出与结果回传
- **feat**: 工作流关键步骤现可自动落文件到 OpenClaw 工作区，目录统一为 `workflows/<channel>/<user>/<shortId>/`
- **feat**: 工作流详情页新增产出文件区，支持文件列表、最终结果标识、回传状态、失败原因、预览、下载、单文件重发与批量重发最终文件
- **feat**: QQ 私聊与 QQ 群聊已支持工作流完成后自动回传最终文件，并在原会话补发“文件已发送”提示
- **feat**: 工作流文件预览已对齐工作区体验，改为弹窗预览，Markdown 支持渲染/源码切换

### 运行时与稳定性
- **feat**: 工作流节点支持 `skill` 参数，若配置 `skill` 将自动加载对应 `SKILL.md` 参与步骤执行
- **fix**: 修复复杂任务在 QQ 路径下重复创建多个工作流实例的问题，并补强 OneBot 会话 ID 归一化
- **fix**: 修复 `wait_user/input` 回复后仍反复要求补充、`approval` 继续后重新阻塞的问题
- **fix**: 修复工作流模型协议兼容问题，支持 `openai-responses` 并兼容旧配置自动回退
- **fix**: 修复 AI 生成模板时 `outputFile` 被错误生成成 `true/false` 导致异常文件名的问题
- **fix**: 优化工作流步骤请求上下文，增加摘要裁剪、超时后精简上下文重试与大步骤输出约束，降低中后段步骤 504 超时概率

### 多通道与日志
- **feat**: 活动日志补充 Workflow / QQ / 飞书 / 企业微信 / OpenClaw 多来源消息流，工作流消息可同步到消息中心
- **fix**: 已取消的工作流实例不再继续出现在消息中心卡片中
