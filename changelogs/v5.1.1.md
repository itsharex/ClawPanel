## v5.1.1 - Agents 可视化配置器升级（官方语义对齐）

### 重点改进
- **feat**: `Agents` 页面 Bindings 编辑器升级为「结构化表单 + JSON 高级模式」
  - 结构化字段覆盖：`channel/sender/peer/parentPeer/guildId/teamId/accountId/roles`
  - 每条规则显示优先级标签（与后端预览语义一致）
  - JSON 模式支持回切结构化（支持表达式会自动保留 JSON 模式）
- **feat**: Bindings 编辑联动 `channels` 配置
  - 自动读取 `channels.<channel>.accounts` 与 `defaultAccount`
  - `accountId` 提供下拉选择与语义提示（留空=默认账号，`*`=全账号兜底）
- **fix**: 路由预览输入字段与后端匹配字段严格对齐，移除无效字段

### 后端语义对齐
- **feat**: `POST /api/openclaw/route/preview` 新增默认账号作用域判断
  - 当 binding 未设置 `match.accountId` 且存在 channel 默认账号时，仅对默认账号生效
  - 预览 trace 增加隐式默认账号不匹配信息，便于排查

### 测试
- **test**: 新增 `TestPreviewRouteChannelOnlyBindingUsesDefaultAccountScope`
  - 验证 channel-only binding 在非默认账号下不会误命中
- 已通过：`go test ./internal/handler`
- 已通过：`npm --prefix web run build`

### 影响说明
- 旧版依赖“channel-only 规则匹配任意账号”的预览习惯将发生变化；新行为按 OpenClaw 官方文档语义执行。
