## v5.1.6 - 修复 QQ 插件安装链路与 NapCat 状态误判

### Bug 修复
- **fix**: 安装 `QQ (NapCat)` 时先确保 `qq` 插件已安装；插件安装失败会显式终止流程，不再继续写入会导致网关报错的 QQ 配置
- **fix**: QQ 通道保存/启用前增加后端校验；若 `QQ 个人号插件` 未安装，直接返回明确错误提示
- **fix**: 插件中心列表默认会主动拉取插件仓库，避免缓存为空时“插件商店打不开 / 无内容”

### NapCat / QQ 状态修复
- **fix**: 当 NapCat 已安装但 `qq` 插件缺失时，系统页不再误显示为正常已安装，而是明确标记为 `缺少插件`
- **fix**: QQ 通道页新增 `QQ 个人号插件未安装` 提示，并提供重新安装 NapCat / 前往插件中心入口

### QQ 通知与开关生效
- **fix**: QQ 通道配置保存后会自动重启 OpenClaw 网关，使 `antiRecall`、`memberChange`、`adminChange`、`banNotice`、`fileUpload`、`pokeReply`、`honorNotice`、`welcome` 等开关立即生效
- **fix**: 无法自动重启时会明确提示需手动重启网关，避免误以为开关无效

### 验证
- 已通过：`go test ./...`
- 已通过：`cd web && npm run build`
