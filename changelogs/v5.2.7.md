## v5.2.7 - QQ 通道稳定性修复与插件安装体验改进

### QQ (NapCat)
- 收紧 QQ 自动接管条件，只有用户明确启用/安装过 QQ 通道，或配置里本来就有 QQ 项时，面板才会继续维护 `channels.qq` 与插件状态，避免未启用时被默认写入 `openclaw.json`
- 为 Docker 版 NapCat 增加账号级 `onebot11_<uin>.json` 网络配置自修复，降低“容器运行中但 WS / HTTP 不可用、登录后掉线”的问题
- 新增“删除 QQ 通道”按钮，可一键删除 NapCat、卸载 `qq` 插件、清空 `openclaw.json` 中所有 QQ 相关配置并重启网关

### 通道安装体验
- 对 Telegram、Discord、Slack、QQ 官方机器人、钉钉、企业微信、飞书等插件型通道，通道页支持直接一键安装，不再必须跳转插件中心
- QQ (NapCat) 仍保留独立安装链路，继续由面板负责 NapCat 环境、登录与诊断

### 插件仓库
- 修复插件镜像仓库缺失 `registry.json` 导致部分用户刷新后只能看到旧插件列表的问题
- 插件仓库刷新新增 Gitee 回退源，降低 GitHub 不可达时的失败概率

### 运行状态
- 只要检测到 OpenClaw 正在运行且网关在线，就统一视为运行正常，不再额外显示“由外部管理”的告警
